# Cppcheck 是什麼？給非技術同事聽得懂的版本

## 1️⃣ 什麼是「寫程式」？

寫程式就是給電腦下一系列命令，讓它照著步驟做事。

最簡單的例子是「說你好」：

- 程式語言告訴電腦：「請把『Hello World』印出來。」
- 就像對助理說：「等我講完，你就大聲說：你好。」

電腦不會思考、不會猜意思，只會完全照著你寫的執行。

## 2️⃣ 重複做事：迴圈（loop）

「從 1 數到 10」對人類很簡單，但對電腦是：

- 重複做同一件事
- 必須很精準，不會漏、不會多數

程式語言有個結構叫 loop（迴圈），用來告訴電腦「一直做，一直到…」。

## 3️⃣ 經典笑話：程式語言的 if/else

> 妻子：下班回家時帶一斤包子，如果看到賣西瓜的，就買一個。  
> 工程師：好。  
>  
> 結果回家 → 只帶回一個包子

為什麼？工程師的腦中是程式邏輯：

```c
if (看到賣西瓜的人) {
    買一個;
}
else {
    買一斤包子;
}
```

程式語言的世界「沒有上下文、沒有常識、沒有推論」。

## 4️⃣ 笑話背後的真實問題

這叫做：文字缺乏主體 → 程式也會找不到主體 → 變數未定義。

如果工程師寫成：

```c
buy(包子數量);  // 但包子數量誰定義？在哪？
```

電腦就會問：「包子數量是什麼？我不知道你在講誰！」

這就是程式中常見的錯誤之一。

## 5️⃣ 會造成程式錯誤的原因（用生活故事解釋）

### （1）❌ 空指標 / 死指標（Null Pointer）

比喻：你叫助理「打電話給張三」，結果手機裡根本沒有張三的聯絡資訊 → 打不了。

在程式世界：

- 空指標 = 我要去找某個東西，但這東西不存在

Cppcheck 在這裡會提醒：

> 你這裡要找的東西是空的，會出事喔！

### （2）❌ 除以零

生活比喻：老闆問「公司今年營收除以人數是多少？」但公司員工人數 = 0？  
不可能計算 → 一定錯！

程式也是：

```c
100 / 0
```

會直接讓程式爆炸。

Cppcheck 能預測出：

> 這裡有機會出現「除以零」，要小心。

### （3）❌ 整數溢位 / 邊界越界

比喻 1：你買的箱子最多放 10 個橘子，你硬塞 20 個 → 爆掉。  
比喻 2：你有 10 個格子的鞋櫃（0~9），你放到「第 20 格」 → 根本不存在 → 出錯。

程式也一樣：記憶體有邊界，不能亂超出格子範圍。

Cppcheck 會說：

> 這個值太大了！可能會溢位。  
> 這裡超出陣列範圍！

### （4）❌ 未初始化變數

比喻：助理問你「要開會嗎？」你說「你自己猜。」  
結果助理可能亂做決定。

程式也是：如果變數沒給值，電腦會拿到亂碼，造成不可預期問題。

Cppcheck 會說：

> 這個變數你沒有先設定就用，會出事！

## 6️⃣ Cppcheck 是什麼？

Cppcheck 是一個「靜態程式碼分析工具」。  
它不需要執行程式，就能檢查程式碼中可能危險的問題。

用非技術語言解釋，Cppcheck 就像：

- 程式碼的「文法老師」
- 工程師的「安全駕駛系統」
- 寫程式的「事前保險」

它能在「程式還沒跑」之前就告訴你：

- 這裡有陷阱
- 這裡可能會 crash
- 這裡可能影響硬體

## 7️⃣ 為什麼 Cppcheck 對公司很重要？

用三句話講清楚：

1. 讓產品更安全：避免機器卡死、APP 當機、硬體出錯
2. 減少維修成本：越早發現錯誤 → 修起來越便宜
3. 保護品牌：品質越高，越不會出現讓客戶抱怨或退貨的問題

## 8️⃣ 最後：把話講給任何人能懂的版本

寫程式就像寫指令給完全不會思考的機器人。  
程式的任何小錯誤，都可能讓機器人做出奇怪的事。  
而 Cppcheck 就像一個「事前校對」工具，  
能在機器人真正執行命令前，就先幫你抓錯誤。
